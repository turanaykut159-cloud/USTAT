## [6.2.0] — 2026-04-18

### Fixed
- #293 — **TopBar initial loading mask** (`desktop/src/components/TopBar.jsx`, C1 Yeşil Bölge UX). TopBar `<App>` içinde Routes dışında global tek instance'tı; sayfa geçişinde state korunuyordu ama ilk mount'ta `getStatus`/`getAccount` polling tamamlanana kadar (~1-2sn) fallback değerler (`V6.0`, `PASIF`, `0,00`) gösteriliyordu — kullanıcı yanlış izlenim ediniyordu. `initialLoading` flag'i artık `dispVersion`, `dispPhaseLabel`, `dispPhaseClass` ve metric değerleri için em-dash maskesi uyguluyor. İlk fetchData() döndüğünde gerçek değerler anlık değişir. Bug değil, UX iyileştirmesi.
- #292 — **PRİMNET grid: trail sonrası geçmiş stop seviyeleri yanlışlıkla "STOP" gösteriyordu** (`desktop/src/components/PrimnetDetail.jsx`, C2 Yeşil Bölge). Trailing aktifleşip slPrim entry'ye doğru kaydığında, geçmiş stop seviyeleri (SELL'de +1.50, +1.00 gibi) hâlâ `STOP` etiketi taşıyordu — semantik hata: pozisyon zaten aktif slPrim'de (+0.50) kapanırdı, üst seviyelere ulaşılamazdı. Eski koşul `prim ≤ entryPrim + trailingDist + 0.25` initial state için tasarlanmıştı, trail sonrası aralığa düşen tüm satırları yanlış STOP işaretliyordu. Yeni mantık: `isCurrentStop` (canlı slPrim'e en yakın satır) STOP'un birincil kapısı, üstündekiler `DIŞARDA`. SL henüz broker'dan okunmadıysa initial stop pozisyonuna fallback (`!hasValidSl && isStop`). BUY/SELL dalları simetrik düzeltildi. Critical flows 119/119 PASS, build 0 hata. Detay: `docs/2026-04-29_session_raporu_primnet_stop_etiket_duzeltme.md`.
- #291 — **Manuel işlem auto-fill SL/TP kaldırıldı** (`engine/manuel_motor.py`, C1 Yeşil Bölge). Modül docstring ("Kullanıcının SL/TP'si aynen korunur") ile kod çelişiyordu: kullanıcı SL/TP vermese de ATR×1.5/2.0 ile zorla türetiliyor, MT5 `modify_position` stops_level uyuşmazlığı yüzünden fail olunca `report_unprotected_position` yanlış alarm tetikliyordu (2026-04-24 F_AKBNK ticket 8050696123 olayı). Auto-fill bloğu (374-387) kaldırıldı, `sl/tp is None or < 0` → `0.0` normalizasyonu eklendi; mevcut `if sl > 0 and tp > 0` koşulu SL/TP verilmediğinde modify_position çağrısını doğal olarak atlatıyor, `calculate_risk_score` zaten `trade.sl > 0` guard'ı ile SL yoksa `sl_risk=yellow` gösteriyor. 5/5 manuel+unprotected critical_flows test PASS.
- #290 — **Baseline güncellemesi `monthly_paused` bayrağını ve aktif kill-switch'i sıfırlar** (`api/routes/settings.py`, C2 Green Zone, test sürecinde).
